# 06 — Execution Planner

## 1. Главная задача

Execution Planner — сердце продукта. Преобразует маркетинговый анализ в ежедневные действия.

**Клиент не получает маркетинговую стратегию. Клиент получает: миссия → задачи → инструкции.**

## 2. Вход и выход

**Вход:** `final_data.json` (результаты 27 блоков Intelligence Layer)

**Выход:** `execution_view_model.json`

## 3. Фазы 30 дней

### Phase 1 — Setup (Дни 1–3)

- **Цель:** Подготовить систему
- **Задачи:** Упаковка, позиционирование, подготовка материалов, контент
- **Запрещено:** Реклама

### Phase 2 — Content (Дни 4–10)

- **Цель:** Собрать первые сигналы рынка
- **Задачи:** Посты, Stories, Reels, комментарии, сообщения

### Phase 3 — Traffic (Дни 11–20)

- **Цель:** Получить трафик
- **Каналы:** VK, Telegram, Яндекс
- **Только тесты**

### Phase 4 — Scale (Дни 21–30)

- **Цель:** Усиление рабочих связок
- **Задачи:** Масштабирование, повтор лучших материалов, ретаргетинг

## 4. Генерация миссий

Каждая неделя имеет миссию:

- **Неделя 1:** Собрать первые реакции
- **Неделя 2:** Получить первые лиды
- **Неделя 3:** Получить первые продажи
- **Неделя 4:** Усилить лучшие связки

## 5. Правило количества задач

- **Максимум:** 5 задач в день
- **Минимум:** 2 задачи
- **Оптимум:** 3 задачи

Причина: чтобы клиент не бросил.

## 6. Типы миссий

| Тип         | Назначение          |
| ----------- | ------------------- |
| **Setup**   | Настройка системы   |
| **Content** | Публикация контента |
| **Ads**     | Реклама             |
| **Sales**   | Продажи             |
| **Review**  | Аналитика           |

## 7. Структура миссии

```json
{
  "day": 1,
  "title": "string",
  "why": "string",
  "steps": ["string"],
  "ready_text": "string",
  "metric": "string",
  "success_threshold": "string",
  "if_success": "string",
  "if_fail": "string"
}
```

## 8. Главное правило Planner

**Запрещено:** «Напишите пост»

**Разрешено:** «Откройте Telegram. Создайте пост. Вставьте текст ниже. Опубликуйте. Проверьте просмотры через 24 часа.»

## 9. Алгоритм выбора задач

Приоритет: Контент → Продажи → Реклама → Оптимизация.

Никогда наоборот.

## 10. Контентная ротация

Запрещено: 10 Reels подряд.

Система обязана чередовать: Reels → Stories → Post → Reels → Post → Stories.

## 11. Archetype Rotation

Список архетипов: `case`, `review`, `tour`, `mistake`, `faq`, `comparison`, `objection`, `checklist`, `before_after`, `behind_the_scenes`

Запрещено использовать один archetype подряд.

## 12. CTA Rotation

Минимум 10 CTA-механик.

Запрещено: «Напишите в директ» 30 раз подряд.

## 13. Логика рекламы

Каждая рекламная задача содержит:

- Что запустить
- Где (площадка)
- На кого (аудитория)
- Бюджет (500–1000 ₽ тест)
- Когда остановить
- Когда масштабировать

**Budget Ladder:**

```
Тест: 500 ₽ → если работает: 1000 ₽ → если работает: 2000 ₽ → 5000 ₽+
```

## 14. Логика продаж

Каждый день: либо новый лид, либо старый лид, либо клиент получает касание.

Нельзя генерировать продажи только через контент.

## 15. KPI Logic

Каждая миссия обязана иметь:

- Success (успех)
- Warning (предупреждение)
- Fail (провал)

Пример: Успех: 20 подписчиков, Предупреждение: 10, Провал: 3.

## 16. Decision Tree

- **Success →** Усилить
- **Warning →** Оставить
- **Fail →** Изменить

## 17. Anti-Repeat Engine

Planner обязан проверить последние 10 миссий.

Нельзя повторять: Hook, CTA, Angle, Archetype ближе 7 материалов.

## 18. Why Engine

Каждая миссия отвечает: «Почему это делаем?»

Не: «Потому что контент полезен.»
А: «Проверяем гипотезу, что начинающие эксперты боятся выглядеть непрофессионально.»

## 19. Daily Load Validator

Время выполнения дня: максимум 90 минут.

Если больше → FAIL.

## 20. Client Simplicity Validator

Каждая миссия проверяется: «Поймёт ли это владелец бизнеса без маркетингового опыта?»

Если нет → REWRITE.

## 21. Planner Output

```json
{
  "missions": 137,
  "days": 30,
  "levels": [],
  "achievements": [],
  "summary": ""
}
```

## 22. Stages Execution Planner

| Stage | Название                   | Функция                                   |
| ----- | -------------------------- | ----------------------------------------- |
| 1     | Mission Generator          | Создаёт миссии (цели на период)           |
| 2     | Task Generator             | Создаёт конкретные действия               |
| 3     | Content Action Builder     | Преобразует материалы в задания           |
| 4     | Advertising Action Builder | Создаёт рекламные задачи                  |
| 5     | Sales Action Builder       | Создаёт задачи продаж                     |
| 6     | Metrics Builder            | Привязывает KPI к задачам                 |
| 7     | Decision Builder           | Создаёт логику «если успех / если провал» |
| 8     | Gamification Builder       | Создаёт игровой слой                      |

## 23. Success Criteria

Execution Planner считается успешным если клиент:

1. Открыл HTML
2. Увидел День 1
3. Понял что делать за 30 секунд
4. Сделал действие без звонка маркетологу
