# 07 — HTML Dashboard

## 1. Главная цель

HTML Dashboard — персональный маркетинговый кабинет владельца бизнеса.

**Не отчёт. Не стратегия. Не PDF.**
После открытия клиент должен за 30 секунд понять: «Что делать прямо сейчас?»

## 2. Главный принцип

Весь кабинет строится вокруг ДЕЙСТВИЙ, а не информации.

Плохо: «Ваш аватар: женщина 35 лет.»
Хорошо: «Сегодня публикуем контент для: женщины 35–45 лет, малый бизнес, боится потерять клиентов.»

## 3. Требования к HTML

- **Offline:** без сервера, без CDN, без npm
- **Открытие:** двойным кликом
- **Mobile First:** минимальная ширина 390px
- **Браузеры:** Chrome, Edge, Safari, Firefox
- **Никаких внешних запросов**

## 4. Архитектура HTML

```
┌─────────────────────────────┐
│ HEADER                      │  Название компании, день, фаза
├─────────────────────────────┤
│ XP / LEVEL / STREAK         │  Прогресс, уровень, серия
├─────────────────────────────┤
│ TODAY MISSION               │  Главная миссия дня
├─────────────────────────────┤
│ TABS                        │  Навигация
├─────────────────────────────┤
│ CONTENT AREA                │  Карточки заданий
└─────────────────────────────┘
```

## 5. Вкладки

Обязательные вкладки:

- **Сегодня** — текущий день, задачи, CTA, KPI (открывается по умолчанию)
- **30 дней** — весь маршрут
- **Контент** — посты, Stories, Reels
- **Реклама** — все рекламные задачи
- **Продажи** — все скрипты продаж
- **Клиенты** — для кого делается маркетинг
- **Метрики** — KPI по каждому действию
- **Почему это работает** — короткое объяснение логики

## 6. Экран «Сегодня»

Самый важный экран. Показывает:

- Текущий день (День 7 из 30)
- Цель дня
- Задачи дня
- Статус каждой задачи
- Кнопки: ✅ Сделано, 🔄 Переделать, ❌ Не сработало

## 7. Карточка задания

Каждая задача имеет единый формат:

- Название
- Почему важно
- Время выполнения
- Сложность
- Пошаговая инструкция
- Готовый текст
- CTA
- Метрика
- Что делать дальше (если успех / если провал)

## 8. Контентный блок

Раздел «Контент» содержит:

- **Посты:** заголовок, текст, хештеги, CTA, кнопка «Копировать»
- **Stories:** 3 истории
- **Reels:** Hook, Кадры 1–4, озвучка, текст на экране, CTA

## 9. Рекламный блок

Для каждой кампании: площадка, цель, аудитория, креатив, бюджет, KPI, когда остановить, когда масштабировать.

## 10. Блок продаж

Скрипты: первый ответ, ответ на цену, ответ на сомнения, дожим, повторное касание. С кнопкой «Копировать».

## 11. Геймификация

- **Уровни:** Новичок → Контент-Машина → Генератор Лидов → Мастер Продаж → Маркетинг Командир
- **XP:** +10 (задача), +15 (контент), +20 (реклама), +30 (продажи), +100 (первая продажа)
- **Streak:** ежедневная серия (3/7/14/30 дней → бонусы)
- **Достижения:** первый пост, первый Reels, первый лид, первая продажа

## 12. Progress Widget

Всегда сверху: ████████░░░░ 43%. Выполнено: 39, осталось: 51.

## 13. Local Storage

Единый ключ: `marketing_os_dashboard_v1`

Структура:

```json
{
  "xp": 0,
  "level": "",
  "streak": 0,
  "completedMissions": {},
  "failedMissions": {},
  "achievements": []
}
```

После перезагрузки состояние восстанавливается.

## 14. JavaScript Modules

```
Dashboard
├── StateManager        # Управление состоянием
├── StorageManager      # LocalStorage
├── ProgressEngine      # Прогресс
├── XPEngine            # Опыт
├── LevelEngine          # Уровни
├── AchievementEngine    # Достижения
├── MissionEngine        # Миссии
├── ContentRenderer      # Контент
├── AdsRenderer          # Реклама
├── SalesRenderer        # Продажи
├── FilterEngine         # Фильтры
├── SearchEngine         # Поиск
└── PDFEngine            # Экспорт в PDF
```

## 15. Boot Sequence

При открытии HTML:

```
load JSON → load state → restore progress → calculate XP → render dashboard
```

## 16. Умный выбор дня

При открытии проверяется первый незавершённый день. Не всегда День 1.

## 17. Anti-Abandonment

Если клиент 3 дня ничего не делал:
«Вы остановились на Дне 12. Сегодня рекомендуем выполнить только одну задачу и вернуться в ритм.»

## 18. Кнопка «Копировать»

На каждом тексте: пост, CTA, скрипт, объявление.
Использует `navigator.clipboard` с fallback на `textarea` + `execCommand` для офлайн-режима.

## 19. Поиск и фильтры

- Поиск по контенту, CTA, аватарам, дням
- Фильтры: день, площадка, аватар, тип контента

## 20. Экспорт в PDF

Кнопка «Скачать PDF». Использует `window.print()` + `@media print` CSS.

## 21. Мобильная вёрстка

При `< 768px`: 1 колонка, табы — горизонтальный скролл, Today Screen всегда 1 колонка.

## 22. Desktop Layout

2–3 колонки для контента. Today Screen всегда 1 колонка.

## 23. Dark/Light темы

Сохранение выбора темы в localStorage.

## 24. Error Recovery

Если localStorage повреждён — сброс состояния без потери данных проекта.

## 25. UI Success Criteria

Клиент за 30 секунд должен понять:

- Что делать сегодня
- Что публиковать
- Что снимать
- Что писать
- Что измерять
- Что делать дальше
